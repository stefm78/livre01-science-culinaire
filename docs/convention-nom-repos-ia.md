# Convention de Nomination des Repositories IA

## Préfixe Officiel

Tout dépôt généré par l'automatisation basée sur ia-activation et Projet-template doit porter le préfixe obligatoire suivant :

`ia-driven-...`

### Exemples
- `ia-driven-livre-recettes`
- `ia-driven-formation-gestion-projet`
- `ia-driven-analyse-marche`

## Règle d'application
- Le nom du dépôt doit toujours commencer par `ia-driven-`, suivi d'un descriptif clair du projet ou de la solution automatisée.
- Toute configuration d'automatisation GitHub Actions, scripts ou documentation associée doit vérifier et appliquer cette règle lors de la création d'un nouveau repo.
- Les projets existants créés avec un autre préfixe doivent être progressivement renommés pour garantir l'uniformité, la traçabilité et la conformité IA.

Cette convention garantit la visibilité, la recherche et l'audit aisé de l'origine IA de chaque dépôt généré automatiquement.
